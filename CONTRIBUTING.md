# 贡献指南

感谢你考虑为OpenClaw项目做出贡献！

## 🎯 如何贡献

### 1. 报告问题
- 使用GitHub Issues报告bug或请求功能
- 描述清晰的问题现象和复现步骤
- 标注相关的项目名称

### 2. 提交代码
1. Fork项目仓库
2. 创建功能分支 (`git checkout -b feature/amazing-feature`)
3. 提交更改 (`git commit -m 'Add some amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 创建Pull Request

### 3. 改进文档
- 修复文档错误
- 添加使用示例
- 翻译或本地化文档

## 📋 开发规范

### 代码风格
- 遵循PEP 8 (Python) 或相应语言规范
- 使用有意义的变量和函数名
- 添加必要的注释

### 提交信息
- 使用清晰的提交信息
- 格式: `类型: 描述`
- 类型: feat, fix, docs, style, refactor, test, chore

### 测试要求
- 新功能需要添加测试
- 确保现有测试通过
- 测试覆盖率不应降低

## 🏗️ 项目结构

### 已完成项目
- `openclaw-session-memory-solution/` - 生产就绪
- `multi-ai-platform-manager/` - 基础框架

### 开发流程
1. 查看`PROJECT_STATUS.md`了解项目状态
2. 选择感兴趣的项目或功能
3. 在Issue中讨论实现方案
4. 开始开发并提交PR

## 🔧 开发环境

### 要求
- Python 3.8+
- Git
- Docker (可选)

### 设置
```bash
# 克隆项目
git clone https://github.com/your-username/projects_openclaw.git

# 安装依赖
cd projects_openclaw
pip install -r requirements.txt
```

## 📞 沟通

### 讨论渠道
- GitHub Issues - 问题讨论
- GitHub Discussions - 功能讨论
- Pull Requests - 代码审查

### 行为准则
- 尊重所有贡献者
- 建设性讨论
- 包容性语言

## 🏆 认可贡献

所有贡献者将被列在项目的贡献者列表中。
